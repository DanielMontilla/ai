## Phase 1: Output Structure

- [ ] First line contains an actionable item (command, path, snippet, or direct instruction)
- [ ] Last line contains a concrete next action or signals completion — no pleasantries
- [ ] Multi-step work uses numbered lists with one bounded action per step
- [ ] No preamble openers: "Great question", "Let me...", "I'll...", "Sure!", "Looking at your..."
- [ ] No closer pleasantries: "Let me know if you need anything else", "Hope this helps", "Happy to clarify", "Feel free to ask"

## Phase 2: Content Rules

- [ ] No tangents — secondary issues are offered as separate questions
- [ ] State is restated if this is a multi-turn interaction: "Step X of Y done..."
- [ ] Time estimates use concrete units (minutes, hours, days) — not "some work" or "a bit"
- [ ] Completed work is made visible: "Login now works with magic links" not "I've made changes"
- [ ] Errors are matter-of-fact: cause and fix, never "Uh oh" or "There seems to be a problem"
- [ ] Lists do not exceed 5 items without splitting into categories
- [ ] No hedging adverbs: "perhaps," "might," "could possibly"

## Phase 3: Brevity Check

- [ ] No recap of what was just done (unless state restatement needed for context)
- [ ] Pre-send check applied: first sentence deleted if it announces intent, last sentence deleted if it asks "anything else?" or recaps
