## Expected Location

The review subagent writes the full review to `.agents/reviews/<feature-name>/<code>.md` with:
- Review Metadata (Target, Review Type, Iteration, Isolation, Max Attempts)
- Per-severity Findings (Critical → Nit) with stable `F<n>` IDs, Status, Attempts, and `### Discussion` threads
- Summary counts (Open, In Review, Resolved, Won't Fix, Escalated)

## Orchestrator Actions

After the review is written:
- [ ] Human reviews each finding (accept, defer, or dismiss)
- [ ] Accepted findings become remediation tasks (`type: defect` or `execution`) with `finding-ref: <F<n>>` in TASK.md frontmatter
- [ ] Dismissed findings noted in the finding's `### Discussion` thread as a `[Human]` turn
- [ ] Remediation tasks added to FEATURE.md task table
